### `/api/account/`

- `/sign_in`

  `POST` - запрос.

  Параметры:
  - `email` — электронная почта пользователя.
  - `pwdHash` — SHA-512 хэш пароля. Длина — 128 символов.

  Ожидаемое поведение:
  - Поиск пользователя по email, проверка пароля через `argon-cffi.verify`.
  - При ошибке: JSON с сообщением "Неправильный логин или пароль", код `404 Not Found`.
  - При успехе: создаются `access` и `refresh` JWT-токены, возвращаются с профилем (имя, почта, телефон, дата регистрации и рождения). Ответ: `200 OK`.
  - При ошибке формата запроса: `400 Bad Request`.

- `/sign_up`

  `POST` - запрос.

  Параметры:
  - `username` — имя пользователя.
  - `email` — эл. почта.
  - `phoneNumber` — номер телефона.
  - `birthdayDate` — дата рождения пользователя.
  - `pwdHash` — SHA-512 хэш пароля. Длина — 128 символов.

  Ожидаемое поведение:
  - Проверка уникальности email. При совпадении — ошибка `409 Conflict`.
  - Генерация соли, хеширование пароля через `argon2`, сохранение пользователя, генерация JWT. Ответ: `201 Created`.
  - При ошибке валидации — `400 Bad Request`.

- `/sign_out`

  `GET` - запрос.

  Параметров нет.

  Ожидаемое поведение:
  - Инвалидация JWT токенов.
  - Ответ: `200 OK`.
  - При отсутствии авторизации — `401 Unauthorized`.

- `/profile`

  `GET` - получение данных.
  `PUT` - обновление данных.

  Параметры (для PUT):
  - `username` *(опц.)* — имя пользователя.
  - `phoneNumber` *(опц.)* — номер телефона.
  - `paymentMethods` *(опц.)* — JSON-массив (напр. карты, СБП и пр.)
  - `deliveryAddress` *(опц.)* — адрес доставки.

  Ожидаемое поведение:
  - Валидация сессии.
  - Обновление профиля. Ответ: `200 OK`.
  - При отсутствии авторизации — `401 Unauthorized`.

### `/api/orders/`

- `/place_order/`

  `POST` - запрос.

  Параметры:
  - `userId` — ID клиента.
  - `cart` — JSON-массив:
    ```
    [
        {
            "pizzaId": 1,
            "chosenIngredients": {
                "remove": ["томат"],
                "add": ["бекон"]
            }
        },
        ...
    ]
    ```
  - `adressId` — ID адреса доставки.
  - `time` *(опц.)* — время или интервал доставки.

  Ожидаемое поведение:
  - Валидация сессии.
  - Проверка состава заказа и ингредиентов.
  - Расчёт цены (удаления не удешевляют).
  - Сохранение заказа: `created`, `isPaid=False`, `isCancelled=False`.
  - Ответ: ID заказа, `201 Created`.
  - Ошибка в данных — `400 Bad Request`.

- `/pay`

  `POST` - запрос.

  Параметры:
  - `orderId` — ID заказа.
  - `paymentMethod` — строка (напр. "Карта", "СБП").

  Ожидаемое поведение:
  - Валидация сессии.
  - Обновление `isPaid=True`.
  - Ответ: `200 OK`.
  - При ошибке — `400 Bad Request`.

- `/cancel_order`

  `GET` - запрос.

  Параметры:
  - `orderId` — ID заказа.

  Ожидаемое поведение:
  - Валидация сессии.
  - Установка `isCancelled=True`.
  - Ответ: `200 OK`.

- `/status?orderId=?`

  `GET` - запрос.

  Ожидаемое поведение:
  - Валидация сессии.
  - Возвращает статус: `created`, `confirmed`, `delivering`, `delivered`, `cancelled`.
  - Ответ: `200 OK`.

- `/history`

  `GET` - запрос.

  Ожидаемое поведение:
  - Валидация сессии.
  - Возвращает список заказов с деталями:
    ```
    [
        {
            "orderId": {{UUID}},
            "isPaid": true,
            "isCancelled": false,
            "totalPrice": 830,
            "timestamp": "2025-05-08T17:30",
            "location": "ул. Пушкина, дом Колотушкина",
            "items": [
                {
                    "pizzaName": "Маргарита",
                    "addedIngredients": ["бекон"],
                    "removedIngredients": ["томат"]
                },
                ...
            ]
        }
    ]
    ```

### `/api/addresses`

- `GET` - список адресов пользователя.

  Ожидаемое поведение:
  - Валидация сессии.
  - Ответ: `200 OK`.
    ```
    {
        adresses: [
            {
                id: "",
                city: "",
                street: "",
                houseNumber: "",
                appartmentNumber: "",
                isDefault: "",
                coordinates: "широта/долгота"
            },
            ...
        ]
    }
    ```

- `PUT` - добавление нового адреса.

  Параметры:
  - `address` — строка.

  Ожидаемое поведение:
  - Валидация сессии.
  - Сохранение в БД. Ответ: `201 Created`.

- `DELETE` - удаление адреса.

  Параметры:
  - `addressId` — ID адреса.

  Ожидаемое поведение:
  - Валидация сессии.
  - Удаление. Ответ: `200 OK`.

- `UPDATE` (или `PATCH`) - редактирование адреса.

  Параметры:
  - `addressId` — ID адреса.
  - `newAddress` — JSON-объект:
    ```
    {
        city: "",
        street: "",
        houseNumber: "",
        appartmentNumber: "",
        isDefault: ""
    }
    ```

  Ожидаемое поведение:
  - Валидация сессии.
  - Обновление в БД. Ответ: `200 OK`.

  Ожидаемое поведение:

  - Удаление. Ответ: `200 OK`.
