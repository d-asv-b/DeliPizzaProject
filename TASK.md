
Оценка 3:

Цель: Создать базовую структуру веб-приложения (MVP) для заказа и доставки пиццы.

Функциональные требования:
1. Управление аккаунтом:
   - Регистрация с хешированием пароля для безопасности.
   - Вход в систему с проверкой учетных данных.
2. Оформление заказа:
   - Стандартное оформление заказа с выбором предложенных вариантов пиццы.
3. Выбор времени доставки:
   - Возможность выбора времени доставки заказа.
4. История заказов:
   - Просмотр истории сделанных заказов.

Оценка 4: 

Цель: Расширить функционал веб-приложения, добавив персонализацию и улучшенное взаимодействие с пользователем.

Функциональные требования:
1. Настройка ингредиентов:
   - Возможность добавления и удаления ингредиентов при заказе пиццы.
2. Система уведомлений:
   - Уведомления о статусе заказа (подтверждение заказа, отправка на доставку, доставка).
3. Предпочтения пользователя:
   - Сохранение предпочтений пользователя при регистрации для упрощения будущих заказов.
4. Подробная история заказов:
   - Отображение детальной информации по каждому заказу, включая состав пиццы, время доставки и место назначения.

Оценка 5: 

Цель: Добавить продвинутые функции для улучшения пользовательского опыта и обеспечить надежное хранение данных.

Функциональные требования:
1. Интеграция с SQLite:
   - Хранение всех пользовательских и заказных данных в базе данных SQLite (не забываем про безопасность и хеширование).
2. Личный кабинет пользователя:
   - Редактирование персональных данных, включая методы оплаты и контактную информацию.
3. Выбор интервала доставки:
   - Возможность выбора интервала доставки с учетом текущей нагруженности сервиса.
4. Интеграция с картами и геолокацией:
   - Возможность выбора место доставки на карте
